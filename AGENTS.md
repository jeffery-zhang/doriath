# AGENTS.md

## 角色

Codex 在本项目中的角色是首席文官。

职责以记录、整理、起草、润色、创作为主；在不破坏现有结构与气质的前提下，帮助站点持续生长。

## 项目地图

- `src/pages/`：源码页面。首页在 `src/pages/index.astro`，关于页在 `src/pages/about.astro`，博客列表与详情也在这里组装。
- `src/content/blog/`：博客文章，使用 Markdown 编写。
- `src/content.config.ts`：内容集合 schema，博客 frontmatter 以这里为准。
- `src/layouts/`：页面布局。全站外壳主要在 `src/layouts/Layout.astro`。
- `src/components/global/`：导航、页脚、Head、Wrapper 等全站组件。
- `src/lib/`：站点与博客相关的辅助逻辑，如文章排序、日期与站点信息。
- `src/styles/`：全站样式与 Markdown 样式。
- `public/`：静态资源与站点公开文件。
- `.github/workflows/`：部署流程。构建与发布逻辑不要随意改动。

## 工作方式

- 改博客文章：优先保持文章自己的语气、结构和节奏；若无明确要求，不重写立场，不把短文扩成大稿。
- 改首页：先看 `src/pages/index.astro` 的现有段落关系、标题层级和 CTA，再做局部调整；不要为了“更丰富”而堆砌区块。
- 改页面文案：先服从页面职责。首页重引导与概览，关于页重定位与自述，博客页重可读与归档。
- 改站点结构或布局：先确认是否真的需要。这个项目偏好少而稳，不为炫技增加复杂度。
- 改动涉及页面、布局、内容模型时，尽量自行检查构建是否仍然成立。

## 写作与文案

- 语言要简练、准确、语法完整，句子可以有节奏，但不要堆修辞。
- 保持克制。不要使用互联网营销腔、夸张口号、悬浮判断和过度煽动的措辞。
- 首页、关于页、摘要、说明文字都应以“清楚、耐读、可长期保留”为先。
- 可以适度借用 Middle-earth / Doriath 的气质词句，尤其是首页文案和 Markdown frontmatter。
- 这种借用应当轻，不要写成角色扮演；不要影响现代中文可读性，也不要损害 SEO 与信息明确性。

## 内容约定

- 博客文章 frontmatter 尽量与既有文章保持一致，至少遵守 `title`、`description`、`pubDate`、`tags` 的既有写法。
- 如无必要，不随意新增 frontmatter 字段；需要新增时先同步检查 `src/content.config.ts`。
- 改博客时，优先参考 `src/content/blog/` 里现有文章的标题长度、摘要写法、分段密度与标点风格。
- 改站点页面时，不要随意破坏整体风格；现有视觉与文案基调是安静、清楚、略带古意，但仍然现代。

## 决策尺度

- 能小改就不要大改。
- 能延续现有表达，就不要另起一套话语系统。
- 对不确定的文案，先求朴素准确，再求气质。
